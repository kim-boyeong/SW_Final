# Cafe Kiosk

> 카페 키오스크 주문 과정을 웹 브라우저에서 체험할 수 있는 정적 웹 프로젝트

## 무엇을 하는 프로젝트인가

Cafe Kiosk는 사용자가 카페 키오스크처럼 메뉴를 고르고, 옵션과 수량을 선택한 뒤, 장바구니 확인과 가상 결제를 거쳐 주문번호와 영수증을 발급받는 웹 애플리케이션입니다.

PRD의 요구사항인 메뉴 조회, 카테고리 선택, 옵션 선택, 장바구니 관리, 총 금액 계산, 결제 방식 선택, 영수증 출력 여부 선택, 주문번호 생성을 HTML, CSS, JavaScript만으로 구현했습니다. Flask, DB, 별도 서버 없이 정적 파일만으로 실행됩니다.

## 사용 기술

| 구분 | 내용 |
| --- | --- |
| HTML | 화면 구조, 메뉴 영역, 장바구니, 결제/영수증 모달 |
| CSS | 반응형 레이아웃, 키오스크 UI 디자인, 카드/버튼/모달 스타일 |
| JavaScript | 메뉴 데이터 관리, 옵션 선택, 장바구니 계산, 결제 흐름, 주문번호 생성 |

## 파일 구조

```text
SW/
├── assets/
│   ├── menu/
│   └── qr/
├── index.html
├── styles.css
├── script.js
└── README.md
```

| 파일 | 역할 |
| --- | --- |
| `index.html` | 웹 페이지의 기본 구조와 화면 요소를 정의 |
| `styles.css` | 전체 디자인, 반응형 화면, 버튼/카드/모달 스타일을 정의 |
| `script.js` | 메뉴 선택, 장바구니, 결제, 영수증 기능을 처리 |
| `assets/menu/` | 메뉴 카드에 표시되는 이미지 파일 |
| `assets/qr/` | 카카오페이, 네이버페이 결제 안내 QR 이미지 |
| `README.md` | 프로젝트 설명 문서 |

## 사용 방법

1. `index.html`, `styles.css`, `script.js`, `assets` 폴더를 같은 위치에 둡니다.
2. `index.html` 파일을 브라우저에서 실행합니다.
3. 카테고리에서 메뉴를 선택합니다.
4. 사이즈, 온도, 수량을 선택한 뒤 장바구니에 담습니다.
5. 장바구니에서 수량 변경, 삭제, 전체 비우기를 사용할 수 있습니다.
6. 결제하기를 눌러 결제 수단을 선택합니다.
7. 결제 완료 후 영수증 출력 여부를 선택하고 주문번호를 확인합니다.

| --- | --- | --- |
| 메뉴 조회 | 커피, 논커피, 디저트 카테고리별 메뉴 표시 | 완료 |
| 카테고리 선택 | 사용자가 원하는 메뉴 카테고리 선택 | 완료 |
| 메뉴 선택 | 메뉴 카드의 선택 버튼으로 주문 옵션 창 열기 | 완료 |
| 사이즈 선택 | Small, Medium, Large 선택 및 추가 금액 계산 | 완료 |
| 온도 선택 | HOT, ICE 선택 | 완료 |
| 스무디 ICE 제한 | 스무디는 ICE만 선택 가능 | 완료 |
| 수량 선택 | 1개 이상 수량 선택 | 완료 |

| 장바구니 수정 | 수량 증가, 수량 감소, 삭제, 전체 비우기 | 완료 |
| 총 금액 계산 | 장바구니 항목의 소계와 총액 자동 계산 | 완료 |
| 결제 방식 선택 | 카드 결제, 카카오페이, 네이버페이 선택 | 완료 |
| QR 결제 안내 | 카카오페이/네이버페이 선택 시 QR 안내 표시 | 완료 |
| 영수증 출력 여부 | 결제 후 영수증 표시/숨김 선택 | 완료 |
| 주문번호 생성 | 1001부터 순차 증가 | 완료 |

## 옵션 정책

| 옵션 | 내용 |
| --- | --- |
| 사이즈 | Small 기본 가격, Medium +500원, Large +1,000원 |
| 온도 | HOT 또는 ICE 선택 |
| 예외 | 스무디는 ICE만 선택 가능 |
| 수량 | 1개 이상 선택 가능 |
| 디저트 | 사이즈와 온도 옵션 없이 수량만 선택 |

## 입력 / 출력

| 구분 | 입력 | 출력 |
| --- | --- | --- |
| 메뉴 선택 | 카테고리, 메뉴 버튼 클릭 | 선택한 메뉴의 옵션 모달 표시 |
| 옵션 선택 | 사이즈, 온도, 수량 | 옵션이 반영된 소계 금액 |
| 장바구니 | 추가, 삭제, 수량 변경 | 주문 항목 목록과 총 결제 금액 |
| 결제 | 카드/카카오페이/네이버페이 선택 | 결제 확인 화면과 QR 안내 |
| 영수증 | 출력 예/아니오 선택 | 주문번호, 주문내역, 총 금액 또는 주문번호만 표시 |

## 예시 출력 / 화면 흐름

예시 주문:

```text
아메리카노 (Large, ICE) x 2 = 8,000원
치즈케이크 x 1 = 5,500원
총 결제 금액 = 13,500원
주문번호 = 1001
```

사용자 흐름:

```text
시작 화면
→ 카테고리 선택
→ 메뉴 선택
→ 옵션 및 수량 선택
→ 장바구니 담기
→ 장바구니 확인
→ 결제 방식 선택
→ 결제 완료
→ 영수증 출력 여부 선택
→ 주문번호 발급
```

## 구현하면서 확인한 점

- 장바구니가 비어 있으면 결제를 진행할 수 없도록 처리했습니다.
- 수량은 1개 미만으로 내려가지 않도록 제한했습니다.
- 결제 완료 후 주문번호가 순차적으로 증가하도록 구현했습니다.
- 모바일 화면에서도 가로 스크롤이 생기지 않도록 반응형 레이아웃을 적용했습니다.
